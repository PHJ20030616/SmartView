# Task 2.3 前端登录注册页面设计

## 1. 目标

在 `smartview-web` 中完成可独立演示和测试的登录注册前端功能，包括：

- 使用 Ant Design 实现登录页和注册页。
- 登录成功后保存 Token 和当前用户信息。
- 页面刷新后恢复登录状态。
- Axios 请求自动携带 Bearer Token。
- 已携带 Token 的请求返回 `401` 时清理会话并跳转登录页。
- 未登录访问业务页面时跳转登录页，并保留原目标地址。

本任务仅完成前端页面、认证状态管理和 Mock 认证流程。Spring Boot 真实接口联调属于 Task 2.4。

## 2. 已确认范围

### 2.1 本任务包含

- 登录和注册页面的视觉与交互实现。
- 前端表单校验、提交加载状态和中文反馈。
- 前端 Mock 登录注册服务。
- Token 与用户信息的本地持久化。
- 认证上下文、路由守卫、Axios 拦截器。
- 单元测试、组件测试和前端用户流程测试。

### 2.2 本任务不包含

- 不调用真实 Spring Boot 登录、注册接口。
- 不修改 Spring Boot 或 FastAPI 服务。
- 不在浏览器保存注册密码，也不实现本地用户数据库。
- 不实现刷新 Token、主动续期、多端会话管理或第三方登录。
- 不手工修改 `smartview-web/src/api/generated/` 中的生成文件；该目录只能由生成命令更新。

## 3. 契约与类型

认证字段和返回结构以 `contracts/web-api/openapi.yaml` 为唯一事实来源：

- `RegisterRequest`
- `LoginRequest`
- `LoginData`
- `UserInfo`

注册表单遵循以下契约约束：

| 字段 | 要求 |
| --- | --- |
| 用户名 | 必填，3-32 个字符 |
| 昵称 | 必填，不超过 64 个字符 |
| 邮箱 | 选填，必须符合邮箱格式 |
| 手机号 | 选填，符合 `^1[3-9]\d{9}$` |
| 密码 | 必填，6-128 个字符 |
| 确认密码 | 仅属于前端表单，必须与密码一致，不发送给服务端 |

业务代码只能引用 OpenAPI 生成的请求、响应和用户类型，不在 `features/auth` 中复制跨端 DTO。若当前生成目录尚无认证类型，实施时应先完善前端 OpenAPI 生成命令并重新生成代码；生成目录不可手动修改。

## 4. 页面设计

### 4.1 整体布局

采用已经确认的品牌分栏布局：

- 桌面端左侧为 SmartView 品牌与核心能力说明，右侧为账号表单。
- 移动端隐藏完整品牌侧栏，改为顶部紧凑品牌标识和单列表单。
- 页面直接呈现可操作的登录或注册体验，不增加营销落地页。
- 使用白色、深青绿色、深色正文和少量暖黄色强调，避免单一色调。

### 4.2 Ant Design 组件

页面使用项目已有的 `antd` 和 `@ant-design/icons`：

- `Form`，并使用 `layout="vertical"`。
- `Input`、`Input.Password`。
- `Button`。
- `Checkbox`。
- `Typography`。
- `App.useApp()` 提供消息反馈。
- Ant Design 图标提供字段含义、密码显隐和提交动作提示。

输入框、按钮、校验状态、禁用状态和加载状态均使用 Ant Design 原生行为，不使用手工模拟的 HTML 输入控件。

### 4.3 注册页

注册字段按以下顺序严格竖向排列：

1. 用户名
2. 昵称
3. 邮箱（选填）
4. 手机号（选填）
5. 密码
6. 确认密码

表单提交行为：

1. Ant Design Form 完成前端校验。
2. 调用 Mock 注册服务，展示提交加载状态。
3. Mock 服务返回用户信息，不保存密码。
4. 展示“注册成功，请登录”的中文提示。
5. 跳转 `/login`，并通过路由状态预填刚注册的用户名。

### 4.4 登录页

登录页包含用户名、密码和“保持登录状态”选项。由于本任务的验收要求刷新后仍保持登录状态，“保持登录状态”默认选中；未选中时仅保存到当前页面运行期间的内存状态，刷新后需重新登录。

Mock 登录服务接受通过前端校验的用户名和密码，返回符合 `LoginData` 结构的模拟 Token 和用户信息。登录成功后跳转规则如下：

1. 若路由中存在合法的原目标地址，则返回原目标地址。
2. 否则进入首页 `/`。
3. 原目标地址只允许站内绝对路径，拒绝协议地址或双斜线地址，避免开放重定向。

## 5. 认证模块边界

`smartview-web/src/features/auth/` 按单一职责拆分：

| 模块 | 职责 |
| --- | --- |
| `authStorage.ts` | 读取、写入和清理 Token 与用户会话 |
| `authService.ts` | 定义页面使用的登录注册服务入口；Task 2.3 使用 Mock 实现 |
| `AuthContext.tsx` | 提供当前用户、登录状态、登录和退出操作 |
| `ProtectedRoute.tsx` | 拦截未登录的业务路由并保存原目标地址 |
| `authRedirect.ts` | 校验和构造站内登录回跳地址 |

页面只通过认证上下文和认证服务完成操作，不直接操作 `localStorage`，也不自行拼接认证请求。

## 6. 会话存储

持久会话使用单一 `localStorage` 键保存：

```ts
{
  token: string;
  user: UserInfo;
}
```

设计约束：

- 不保存密码、确认密码或完整表单数据。
- 读取时必须校验数据为对象，且 Token 和必要用户字段存在。
- 存储损坏、JSON 解析失败或浏览器禁止存储时，按未登录处理并清理无效值。
- `AuthProvider` 使用存储值初始化状态，因此刷新页面不会先错误跳转到登录页。
- 未选择“保持登录状态”时，认证模块仅在内存中保存本次会话。

## 7. Axios 请求与响应处理

### 7.1 文件职责调整

- `src/api/http.ts` 保留请求追踪 ID 的创建、持久化和同步辅助函数。
- 新建 `src/api/request.ts`，作为项目唯一 Axios 实例。
- 现有业务模块和后续生成客户端统一复用 `request.ts` 中的实例。

### 7.2 请求拦截器

每次请求执行前：

1. 添加或复用 `X-Trace-Id`。
2. 从认证存储读取当前 Token。
3. Token 存在时设置 `Authorization: Bearer <token>`。
4. Token 不存在时不创建空认证头。

### 7.3 响应拦截器

响应成功或失败时继续同步服务端返回的追踪 ID。

只有同时满足以下条件时才执行全局 Token 失效处理：

- HTTP 状态码为 `401`。
- 失败请求实际携带了 `Authorization` 请求头。

处理步骤：

1. 清理内存和本地持久会话。
2. 记录当前站内路径作为登录后的回跳地址。
3. 若当前不在 `/login` 或 `/register`，携带失效原因跳转到登录页。
4. 防止多个并发 `401` 重复触发跳转。

登录请求本身不携带 Token，因此用户名或密码错误产生的 `401` 只由登录页展示，不触发“登录状态失效”的全局跳转。

## 8. 路由设计

公开路由：

- `/login`
- `/register`

受保护路由：

- `/`
- `/resume`
- `/interview`
- `/report`

受保护业务路由统一包裹 `ProtectedRoute`，再渲染 `MainLayout`。未登录访问时跳转 `/login` 并保存完整站内目标路径。已登录访问 `/login` 或 `/register` 时直接进入合法回跳地址或首页，避免重复登录。

未知路由仍回到 `/`，随后由路由守卫根据登录状态决定最终页面。

## 9. 用户反馈与错误处理

- 所有用户可见文案使用中文。
- 字段格式错误显示在对应输入框下方。
- 提交期间按钮展示加载状态并阻止重复提交。
- Mock 服务异常显示通用中文错误，不暴露堆栈或底层对象。
- 本地存储不可用时仍允许当前页面内登录，但提示刷新后需要重新登录。
- Token 失效跳转完成后，登录页根据失效原因展示“登录状态已失效，请重新登录”，Axios 层不直接依赖 Ant Design 消息上下文。
- 必要中文注释用于说明存储降级、401 判定、开放重定向防护和 Mock/真实服务切换边界。

## 11. 验收映射

| 验收标准 | 设计实现 |
| --- | --- |
| 用户可以从页面完成注册 | 注册页校验、Mock 注册、成功提示和登录页跳转 |
| 用户可以登录并进入系统 | Mock 登录返回 Token，认证上下文更新并跳转业务页 |
| 刷新页面后仍能读取登录状态 | 持久会话由 `AuthProvider` 初始化读取 |
| 未登录访问业务页会跳转登录页 | `ProtectedRoute` 统一保护业务路由 |
| 请求自动携带 Token | `request.ts` 请求拦截器注入 Bearer Token |
| Token 失效跳转登录页 | 仅针对携带 Token 的 `401` 清理会话并跳转 |
